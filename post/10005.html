在苹果系统上安装配置VSCode

发表于2023-01-05|更新于2026-01-05|操作系统
|总字数:827|阅读时长:2分钟|浏览量:

第一节：下载和安装

浏览器访问 VSCode 的官网，点击下图中的Download for macOS：

下载下来的是一个.dmg格式的安装程序：

双击下载下来的安装程序，然后将 VSCode 图标拖入应用程序图标，完成安装。

第二节：自定义设置

2.1：初步设置

VSCode 需登陆两种账号，首先第一种是跟传统软件一样的，登录以后可以同步软件的个性化设置；第二种是选择性登录的，登录了以后可以使用 GitHub Copilot。

第一次打开 VSCode 提示登录的是后者：

我暂时先选择不登录，直接点击右下角的 Continue without Signing In，然后选择颜色主题：

点击 Continue 后，会再次提示登录 GitHub 账号来使用 GitHub Copilot，暂时跳过，点击 Get Started：

这样就进入了 VSCode 的欢迎页：

2.2：语言设置

默认情况下 VSCode 是英文的，可以修改为中文：

安装完成后，右下角会有重启提示，点击右下角的“Change Language and Restart”即可完成：

2.3：登录账号

这里登录的是 2.1 中描述的前者。点击左下角圆形人头图标，再点击“备份和同步设置”，会出现登录按钮：

点击这个登录按钮，有两种选择，我选择使用 Microsoft 账号登录，登录成功后会有提示：

2.4：工作区信任

在 VSCode 中有“受信任区”这么个说法，所谓受信任区，简单理解就是 VSCode 可以访问和编辑的文件目录，假如使用 VSCode 打开非信任区的文件或非信任文件时，总是会给出提示：

可以通过设置来关闭工作区信任：

2.5：字体相关

VSCode 的字体有两种，一是页面 UI 字体，二是编辑区的字体！我们可以通过command和-或+来调整界面 UI 字体大小，而对于编辑区的默认字体，需要打开设置：

还可以通过command和鼠标滚轮的组合来灵活修改编辑区字体。在设置页面搜索Mouse Wheel Zoom：

2.6：Tab Size

在文本编辑器中，尤其是使用 VSCode 编辑代码时，我们一般会使用 Tab 键来执行缩进操作。但是 Tab 键有时候虽然看起来是缩进了 4 个空格，但是它在编码上并不代表 4 个空格。所以这里我们设置一下，设置后的效果就是：每当键入 Tab 时，都能不多不少地正确输入 2 个空格！

第一步：打开设置，搜索Tab Size，在常用设置里将一个制表符等于的空格数设置为 2：

第二步：关闭Detect Indentation。假如不关闭的话，VSCode 会根据文件内容自动推测 Tab 的空格数：

2.7：控制字符

VSCode 的编辑页面默认会隐藏空格等字符（其他编辑器基本也是这样），但我使用它主要是用来看代码的，所以我希望可以在编辑区展示包括空格在内的所有字符！
